# План внедрения фичей (Трекинг времени, Меню, Синхронизация)

Все твои пожелания услышаны. Текущий сервер пока официально признан "Песочницей" (соответствующая запись внесена в `BACKLOG.md`). Ниже план реализации новых фичей с учетом тестирования:

## Proposed Changes

### 1. Трекинг времени ответа + Кнопка "Выходной"
- **`prisma/schema.prisma`**
  Добавим новую модель `DailyCheckin` для хранения аналитики (во сколько ты нажал кнопку и какая емкость выбрана). Поскольку бот будет расширяться для нескольких пользователей, мы сразу заложим колонку `user_id` (куда пока будем писать твой Telegram Chat ID), чтобы потом табличку не пришлось переделывать.
- **`src/infrastructure/adapters/telegram.adapter.ts`**
  Заменим кнопку "30% — ~2 часа" на "🌴 Сегодня выходной" (`capacity_0`).
- **`src/core/application/orchestrators/planning.orchestrator.ts`**
  При получении ответа `capacity` будем создавать запись в таблице `DailyCheckin` (сохраняя текущее время, `new Date()`).
  Если пришел `capacity = 0` (Выходной): бот автоматически перенесет **все** сегодняшние задачи на завтра и сгенерирует радостное сообщение про отдых, не задавая лишних вопросов.

### 2. Системное Меню в Telegram
- **`src/transport/webhooks/telegram.update.ts` (или `adapter`)**
  Добавим метод `onModuleInit`, который при старте приложения будет отправлять Телеграму набор команд с описаниями через API `setMyCommands`. После этого слева от поля ввода в ТГ появится синяя кнопка "Меню" со списком: `/dashboard`, `/today`, `/list`, `/postponed`, `/logs`.

### 3. Синхронизация старых тасок из Jira
- **`src/transport/cron/scheduler.controller.ts`**
  Добавим новый крон, который каждую ночь (например, в 03:00) будет вызывать `taskSyncOrchestrator.syncAllJiraTasks()`. Это гарантирует, что если в Jira что-то изменилось без вебхуков, база всё равно обновится и подтянет все старые задачи.

### 4. Дебаг-команды для мгновенного теста (Developer Sandbox Tools)
- Чтобы тебе не приходилось ждать кронов на сервере, я добавлю скрытые команды в бота:
  - `/test_morning` — моментально запускает утренний опрос.
  - `/test_evening` — моментально делает вечерний чекап.
  - `/sync_all` — принудительно запускает полную синхронизацию Jira с базой (вытаскивает всё).

### 5. Автоматизация миграций БД при деплое
- **`.github/workflows/deploy.yml`**
  Чтобы тебе не приходилось руками обновлять базу данных каждый раз при изменении `schema.prisma` (как сейчас с таблицей DailyCheckin), я изменю команду в твоем деплой-скрипте с `yarn prisma migrate deploy` на `yarn prisma db push`. Теперь структура базы будет автоматически подгоняться под свежий код при каждом коммите!
