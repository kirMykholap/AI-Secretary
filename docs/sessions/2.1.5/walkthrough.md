# Snyk Security CI/CD (Сессия 2.3)

## 📌 Что реализовано:

Мы сделали профессиональное разделение на 2 пайплайна:
1. **CI (Continuous Integration): `.github/workflows/security.yml`**
   - Запускается при каждом коммите/PR (в том числе в `dev` и `main`).
   - Использует `snyk/actions/node`, чтобы просканировать `package.json` и `yarn.lock`.
   - Если находится уязвимость уровня **High** или **Critical** — падает с ошибкой (красный крестик).
   - Все Low/Medium уязвимости собираются и логируются в самом Action + отправляются тебе на дашборд Snyk.io!
2. **CD (Continuous Deployment): `.github/workflows/deploy.yml`**
   - Больше не прослушивает событие `push` напрямую!
   - Вместо этого он дожидается события: `workflow_run: Security Check -> completed`.
   - Перед деплоем он проверяет условие: `if: conclusion == 'success'`. Если Snyk не дал добро, копирование файлов на сервер отменяется.

## 🚀 Как проверить
Я уже запушил этот код в твою ветку `dev`. 

1. Зайди на GitHub в раздел **Actions**. Ты увидишь, что прямо сейчас побежал экшен "Security Check" для твоего нового коммита в dev-ветке!
2. Теперь, чтобы обновить сервер — просто создай PR из `dev` в твою главную ветку (как ты делаешь обычно) и нажми Merge.
3. Github сперва запустит сканирование. А когда оно пройдет успешно — магическим образом автоматически запустится "Deploy to VPS"!
