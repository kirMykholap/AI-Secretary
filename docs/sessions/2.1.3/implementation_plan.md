Этот план описывает шаги по исправлению 6 основных багов, описанных в BACKLOG.md.

## User Review Required

> [!IMPORTANT]
> Баг №2 ("Новый тест") вероятнее всего связан с тем, что в самой базе данных SQLite/Postgres застряла или "подвисла" задача, которая не была удалена из-за сбоя синхронизации. В процессе выполнения мы найдем её и удалим из БД.

## Proposed Changes

### 1. Daily/Evening Check Bug
**Файл:** `src/core/application/orchestrators/planning.orchestrator.ts`
В метод `processEveningCheckup` мы добавим выборку `getOverdueTasks`, чтобы вечерний чек видел не только задачи со сроком на *сегодня*, но и просроченные (вчерашние) задачи, как это сделано для утреннего плана. Сейчас он берет только `getActiveTasksByDueDateRange(today, tomorrow)`, поэтому старых тасок не видит вообще.

### 2. Command Bug (`/logs`)
**Файл:** `src/infrastructure/logger/file.logger.ts`
Во время чтения лог-файла `content.trim().split('\\n')` использует буквальный двойной слэш и 'n' вместо символа новой строки `\n`. Я исправлю все экранирования, чтобы `split` корректно разбивал текст по переносам строки.

### 3. TickTick Reschedule Bug
**Файл:** `src/infrastructure/adapters/ticktick.adapter.ts` и `telegram.adapter.ts`
При переносе задачи мы отправляли в TickTick объект: `{ dueDate: 'новая_дата' }`. Скорее всего API TickTick при частичном обновлении требует поля, либо мы вообще не получаем эту ошибку явно. Будем сначала забирать задачу `getTaskById`, менять в ней `dueDate` и отправлять её обратно целиком (с правильным `projectId`).

### 4. Jira -> TG Sync Bug & Freeze
**Файл:** `src/transport/webhooks/jira-webhook.controller.ts` и `task-sync.orchestrator.ts`
Зависание терминала, вероятно, связано с необработанным промисом или бесконечным циклом/таймаутом при создании задачи, из-за чего поток NodeJS заблокировался. Мы обернем хендлеры в строгие `try/catch` с таймаутами и поправим закрытие задачи в Jira (transition), чтобы оно было бесшовным.

### 5. Text adjustments
**Файл:** `src/transport/webhooks/telegram.update.ts`
Уберем странные названия приоритетов ('Никакой', 'Эх...') и унифицируем тексты, чтобы они выглядели профессиональнее. Косметический рефакторинг текстов кнопок.
