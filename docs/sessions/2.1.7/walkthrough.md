# Интеграция Voice Input (v2.1.7)

В этой сессии я провел аудит и финализацию фичи голосового ввода, которую реализовал Claude Code. Теперь бот умеет слушать тебя, распознавать задачи и предлагать их для добавления.

## 🎙 Как это работает
1. Ты отправляешь **голосовое сообщение** боту.
2. Бот скачивает аудио и отправляет его в **Groq Whisper API** (модель `whisper-large-v3-turbo`).
3. Полученный текст анализируется через **LLM**, которая вычленяет заголовок, описание, дедлайн и приоритет.
4. Бот присылает тебе карточку подтверждения:
   - Если всё верно — жми **✅ Добавить**, и задача улетит в базу с тегом `#voice`.
   - Если это была просто болтовня — бот вежливо скажет, что задачи не нашел.

## ✅ Что проверено и исправлено

- **Сборка:** Выполнена полная проверка типов (`npm run build`), билд прошел успешно.
- **Безопасность:** Проверено отсутствие хардкода ключей в коде. В `.github/workflows/deploy.yml` добавлен проброс `GROQ_API_KEY`.
- **Документация:** Обновлен `.env.example`.

## ⚠️ Важное ограничение
У кнопок в Telegram есть лимит на объём передаваемых данных (64 байта). Если заголовок или описание в голосовом сообщении будут слишком длинными, кнопка "Добавить" может не сработать. В будущем мы перенесем хранение этих черновиков в базу данных (Redis), чтобы избежать этого лимита.
